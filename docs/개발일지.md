## 2026-06-09 — 전체 구현 및 배포

### Phase 1 — 프레임워크 전환 (순수 CSS → Tailwind + 멀티페이지)

초기에 Vite + React 18 + 순수 CSS로 단일 페이지 스크롤 방식으로 구현했으나,
참고 레포(`aebonlee/rest03`)의 구조에 맞춰 전면 재구성.

**기술 결정**
| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| 스타일 | 순수 CSS + CSS 변수 | Tailwind CSS v3 (`darkMode: 'class'`) |
| 라우팅 | 없음 (스크롤 앵커) | React Router DOM v7 · HashRouter |
| 다크모드 | `data-theme` 속성 | `document.documentElement.classList` |
| 구조 | 단일 페이지 | 멀티페이지 + SubPageLayout |

**컬러 팔레트 (5색)**
| 이름 | 값 | 용도 |

### Phase 3 — 버그 수정 및 개선

#### 수정 1. OurServices 카드 정렬 문제
- **증상**: 카드 4개가 좌측 편향 (우측 공백 발생)
- **원인**: `max-w-container` 미적용으로 카드 컨테이너와 헤딩 컨테이너 기준 불일치
- **수정**: `mx-auto max-w-container justify-center` 추가

#### 수정 2. 가로 스크롤 차단
- **증상**: 화면 가로 축소 시 좌측 카드 잘림, 오른쪽으로 스크롤 불가
- **원인**: `justify-center` + `overflow-x-auto` 조합 — flex 중앙정렬 시 양쪽으로 overflow 발생, 왼쪽은 스크롤 불가 영역
- **수정**: 외부 `overflow-x-auto` + 내부 `w-fit mx-auto` flex 이중 구조로 분리

#### 수정 3. 동영상 탭 전환 시 스크롤 미이동
- **증상**: 스크롤 내린 상태에서 탭 전환 시 콘텐츠만 바뀌고 화면 위치 유지
- **원인**: `handleCat` 함수가 상태 변경만 처리
- **수정**: `window.scrollTo({ top: 0, behavior: 'smooth' })` 추가

#### 수정 4. 이모지 제거
- **증상**: 일부 환경에서 이모지 미표시 또는 디자인 이질감
- **수정**: `🌿 🌙 ☀️ ✕ ✓ ▶` 전부 인라인 SVG 아이콘으로 교체

#### 수정 5. 푸터 콘텐츠 섹션과 혼동
- **증상**: 흰 배경 상단 소개 영역이 본문 탭 콘텐츠와 시각적으로 구분 안 됨
- **수정**: 상단·하단 2단 구조 → `bg-neutral-950` 단일 다크 배경으로 통합, 사이트맵 제거

#### 수정 6. OG 이미지 색상 불일치
- **증상**: 짙은 네이비 OG 이미지가 흰 배경 기반 사이트와 색감 괴리
- **수정**: 흰 배경 + 우측 Sky Blue→Eco Green 그라데이션 패널 디자인으로 재설계

### 알아둘 함정

| 항목 | 내용 |
|------|------|
| OG 이미지 포맷 | 반드시 JPG (RGB, 알파 없음) — `.flatten().jpeg()` 필수. 카카오·패들렛 스크래퍼 PNG RGBA 미지원 |
| GitHub Pages base | `vite.config.js`에서 `command === 'build'` 시만 `base: '/rest04/'` 적용 |
| 다크모드 flash | `index.html` `<head>` 인라인 스크립트가 `localStorage` 테마를 즉시 적용 (`classList.add('dark')`) |
| `justify-center` + `overflow-x-auto` | flex 중앙정렬 + 가로 스크롤 동시 사용 시 왼쪽 항목 잘림 — 반드시 이중 div 구조 사용 |
| 패들렛 OG 캐시 | URL 변경 없이 카드 재추가해도 캐시 유지. URL에 처음 보는 fragment(`#/`) 추가 또는 24~48시간 대기 필요 |
| SubPageLayout imageUrl | `headLabel` prop과 별도로 `imageUrl` prop 필요 — About·Services·Contact에서 각각 `subpageImages[key]` 전달 |
