## 이번 작업으로 추가된 강점

### 1. 정적 호스팅에서의 풀 인증·DB
GitHub Pages(정적)에 별도 서버 없이 Supabase JS 클라이언트 + RLS로 **회원 전용 게시판과 로그인**을 구현했습니다. anon 키는 노출돼도 RLS가 데이터를 보호하는 구조를 정확히 적용했습니다.

### 2. DB 레벨 권한 강제 (RLS)
"본인 글만 수정/삭제"를 UI 가드가 아니라 **RLS 정책(`auth.uid() = author_id`)** 으로 강제해, 클라이언트를 우회한 요청도 DB가 거부합니다. insert 정책의 `with check`로 author_id 위조도 차단됩니다.

### 3. 이메일 + 카카오 이중 인증
이메일/비밀번호와 카카오 OAuth를 모두 지원합니다. 카카오 이메일 scope가 비즈앱을 요구하는 제약(KOE205)을 닉네임 scope로 우회해, 추가 심사 없이 동작하도록 설계했습니다.

### 4. 공유 프로젝트 대비 네임스페이싱
테이블에 `rest04_` 접두어를 적용해 하나의 Supabase 프로젝트를 여러 앱이 공유해도 충돌하지 않게 했습니다. (무료 플랜 프로젝트 2개 제한 대응)

## 개선 필요

### 1. 카카오 실연동 검증 미완
코드·scope는 준비됐으나 카카오 디벨로퍼스의 동의항목(닉네임) ON + Redirect URI 등록이 끝나야 실제 로그인이 검증됩니다.

### 2. 이메일 없는 카카오 사용자
닉네임 scope만 받으므로 카카오 가입자는 이메일이 없습니다. 추후 알림 메일 등이 필요하면 비즈앱 전환 후 이메일 동의항목을 추가해야 합니다.

### 3. 게시판 부가 기능 부재
댓글·검색·페이지네이션·조회수가 없습니다(MVP 범위). 글 수가 늘면 목록 페이지네이션부터 보강이 필요합니다.

### 4. Contact ↔ 게시판 이원화
Contact 문의는 여전히 stub이고 게시판과 별개입니다. QnA 게시판으로 통합하면 데이터·운영을 일원화할 수 있습니다.
