## 트러블슈팅

### 1. 카카오 로그인 KOE205 / "설정하지 않은 동의 항목"
- **증상**: "카카오로 시작하기" 클릭 → 카카오 화면에서 `KOE205` 에러. 메시지: "설정하지 않은 동의 항목: account_email, profile_image, profile_nickname"
- **원인**: Supabase(GoTrue)의 카카오 provider는 `account_email`·`profile_image`·`profile_nickname` **3개를 서버에서 강제로 요청**한다. 카카오 앱 동의항목에 이 항목이 활성화돼 있지 않으면(="사용안함") 에러.
- **GoTrue 소스로 확인 (강제 사항)**: `internal/api/provider/kakao.go`에서 기본 scope가 위 3개로 하드코딩돼 있고, 클라이언트가 넘긴 `scopes`는 거기에 **append(추가)만** 된다.
  ```go
  oauthScopes := []string{"account_email", "profile_image", "profile_nickname"}
  if scopes != "" { oauthScopes = append(oauthScopes, strings.Split(scopes, ",")...) }
  ```
  → Supabase 기본 카카오 provider를 쓰는 한 이 3개 요청은 **제거 불가능**. 인가코드 요청(클라이언트 코드)으로는 손댈 수 없다.
- **헛다리(기록용)**: 처음엔 `signInWithOAuth`에 `scopes: 'profile_nickname'`을 줘서 이메일 scope를 빼려 했으나 **무효**(위 append 동작 때문). 해당 옵션은 도로 제거 → 현재 코드가 정답.
- **무관한 것 (헷갈리기 쉬움)**:
  - 앱의 **이메일/비밀번호 회원가입 폼과 무관**. `account_email`은 *카카오 사용자의 이메일을 읽는 카카오 동의항목*이지, 우리 가입 폼 입력칸이 아니다.
  - Supabase **Authentication > Sign In 옵션(Confirm email 등) 토글과도 무관**. scope는 GoTrue에 하드코딩돼 있어 대시보드 토글로 안 바뀐다. (단, Providers > Kakao = Enabled + Callback URL 등록은 별개로 필수)
- **최종 해결**: 카카오 앱을 **비즈앱(개인 개발자)으로 전환**한 뒤 동의항목 3개를 모두 활성화.
  - `profile_nickname` → 필수 동의
  - `account_email` → 필수/선택 동의 (**비즈앱 전환 시에만 활성화 칸이 열림**. 일반 앱은 "설정불가". 사업자번호 없는 개인도 OWNER 본인인증 후 무료 전환 가능)
  - `profile_image` → 선택 동의 ("사용안함"이면 안 됨, 반드시 활성화)
  - Supabase가 요청하는 scope와 동의항목이 일치 → 에러 해소. (2026-06-10 설정 완료)

### 2. 테이블 접두어 도입
- **증상**: 공유 Supabase 프로젝트에서 일반적인 `posts` 테이블명은 다른 앱과 충돌 위험
- **수정**: `rest04_posts`로 변경. `teardown.sql`로 구버전 삭제 후 접두어 스키마 재생성. 코드는 `posts.js`의 `TABLE` 상수 한 곳만 수정

## 알아둘 함정 (이번 작업)

| 항목 | 내용 |
|------|------|
| GitHub Actions secrets | `build` job은 `environment` 미지정 → **Repository secrets**에 넣어야 함 (Environment secrets는 `deploy` job 전용이라 빌드 때 안 잡힘) |
| anon 키 노출 | anon public 키는 번들에 박혀도 안전(RLS로 보호). `service_role` 키는 절대 클라이언트/`.env`에 넣지 말 것 |
| HashRouter + OAuth 콜백 | 카카오 복귀 시 `#access_token=...`이 잠깐 붙음. supabase `detectSessionInUrl`이 읽고 정리하면 정상. `redirectTo`는 `origin + BASE_URL` |
| RLS author_id | `default auth.uid()` + insert 정책 `with check (auth.uid() = author_id)`로 위조 차단. 클라이언트가 보낸 author_id가 본인과 다르면 거부 |
| 회원가입 이메일 인증 | Auth 기본값이 "Confirm email" ON이면 가입 후 메일 링크 전엔 세션 없음. 데모 편의 시 대시보드에서 해제 |

## 전체 연동 과정 (Supabase · DB · Git · 카카오)

처음부터 재현할 수 있도록 전체 흐름을 정리한다. 순서대로 따라가면 로컬·배포 모두 동작한다.

### 전체 구성도

```
[브라우저(React SPA)]
   │  ① 이메일/비번 or ② 카카오 로그인
   ▼
[Supabase Auth]  ──(카카오 OAuth)──▶  [카카오 로그인 서버]
   │                                       ▲
   │  세션(JWT) 발급                        │ 동의항목 일치 필요
   ▼                                       │
[Supabase DB(Postgres) + RLS]      [카카오 디벨로퍼스 비즈앱]
   │  rest04_posts 테이블, author_id=auth.uid()
   ▼
[게시판 목록/상세/작성/수정/삭제]

빌드·배포: GitHub(main push) ─▶ Actions ─(env 주입)─▶ GitHub Pages
```

### STEP 1. Supabase 프로젝트 (계정·프로젝트)

1. [supabase.com](https://supabase.com) 가입 → **New project** 생성 (리전: Northeast Asia(Seoul) 권장)
2. 프로젝트 생성 후 **Project Settings → API** 에서 두 값 확인:
   - **Project URL** (`https://xxxx.supabase.co`)
   - **anon public** 키 (브라우저 노출 OK — RLS로 보호)
   - ⚠️ **service_role** 키는 절대 클라이언트/`.env`/git에 넣지 않는다.

### STEP 2. DB 스키마 + RLS (SQL Editor)

Supabase 대시보드 **SQL Editor** 에서 순서대로 실행:

1. (구버전 있으면) `supabase/teardown.sql` — 접두어 없는 `posts` 정리
   ```sql
   drop table if exists public.posts cascade;
   drop function if exists public.touch_updated_at() cascade;
   ```
2. `supabase/schema.sql` — `rest04_posts` 테이블 + RLS 정책 생성
   - 테이블: `id, category(notice|free|qna), title, content, author_id, author_name, created_at, updated_at`
   - `author_id`는 `default auth.uid()` + FK(`auth.users`)
   - RLS 정책 4종:
     - **select**: `to authenticated` (로그인해야 읽기 — 회원 전용)
     - **insert**: `with check (auth.uid() = author_id)` (본인 id로만)
     - **update / delete**: `using (auth.uid() = author_id)` (본인 글만)
   - `updated_at` 자동 갱신 트리거(`rest04_touch_updated_at`)
3. **Table Editor**에서 `rest04_posts` 생성 + RLS "Enabled" 확인.

### STEP 3. Supabase Auth 설정

1. **Authentication → Providers → Email**: 사용. 데모 편의 시 "Confirm email" OFF(가입 즉시 로그인). 운영은 ON 권장.
2. **Authentication → URL Configuration**:
   - **Site URL**: `https://inorrni.github.io/rest04/`
   - **Redirect URLs**(허용 목록)에 추가:
     - `http://localhost:5173`
     - `https://inorrni.github.io/rest04/`
3. **Authentication → Providers → Kakao**: (STEP 5에서 키 받은 뒤) Enabled + REST API 키 + Client Secret 입력.
   - 이 화면 상단의 **Callback URL**(`https://<project>.supabase.co/auth/v1/callback`)을 복사 → STEP 5에서 카카오에 등록.

### STEP 4. 로컬 환경변수 (.env.local)

```bash
cd rest04
cp .env.example .env.local
```
`.env.local` 에 STEP 1의 값 입력:
```
VITE_SUPABASE_URL=https://xxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOi...
```
- `.env.local`은 `.gitignore` 처리됨(커밋 안 됨). `.env.example`만 git에 올라감.
- `npm run dev` → `http://localhost:5173` 로 동작 확인. (이메일 로그인은 이 단계까지 하면 바로 됨)

### STEP 5. 카카오 로그인 연결 (디벨로퍼스 + 비즈앱)

[developers.kakao.com](https://developers.kakao.com) → 내 애플리케이션:

1. **앱 생성** → **앱 키 → REST API 키** 확보
2. **카카오 로그인 → 활성화 설정**: ON
3. **카카오 로그인 → Redirect URI**: STEP 3의 Supabase Callback URL 등록
   `https://<project>.supabase.co/auth/v1/callback`
4. **카카오 로그인 → 보안 → Client Secret**: 코드 생성 + 활성화 → Supabase Kakao provider에 입력
5. **비즈앱 전환** ([앱 설정] → [비즈니스]):
   - **개인 개발자 비즈앱**으로 전환(사업자등록번호 불필요)
   - 개인정보 보호책임자 정보 입력, 개인정보처리방침 URL: `https://inorrni.github.io/rest04/#/privacy`
6. **카카오 로그인 → 동의항목** (비즈앱이라야 이메일 활성화 가능):
   | 동의항목 | ID | 상태 |
   |---|---|---|
   | 닉네임 | `profile_nickname` | 필수 동의 |
   | 카카오계정(이메일) | `account_email` | 필수 동의 |
   | 프로필 사진 | `profile_image` | 선택 동의 |
   > Supabase(GoTrue)가 이 3개를 강제 요청하므로 모두 활성화돼야 KOE205가 안 난다.

### STEP 6. Git / GitHub Pages 배포

1. **브랜치 작업**: `feat/board-auth-supabase` 에서 개발 → PR #1.
2. **GitHub → Settings → Secrets and variables → Actions → Repository secrets** 에 등록:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
   > `build` job은 environment 미지정이라 **Repository secrets**여야 빌드 때 잡힌다. (Environment secrets는 `deploy` job 전용)
3. `.github/workflows/deploy.yml` 의 build 스텝이 위 secret을 `env`로 주입:
   ```yaml
   - run: npm run build
     env:
       VITE_SUPABASE_URL: ${{ secrets.VITE_SUPABASE_URL }}
       VITE_SUPABASE_ANON_KEY: ${{ secrets.VITE_SUPABASE_ANON_KEY }}
   ```
4. `main` 에 머지 → Actions가 빌드 후 GitHub Pages 자동 배포.
